## 4. 버튼 스타일

- **기본 액션 버튼(저장·카톡전송) = 크림 바탕 + 얇은 골드 테두리 (B안 톤).** 정제·고급 느낌이 확정안.
  - **Why:** 골드 채움/카카오 노랑(#FEE500)/딥그린 등 여러 색을 거쳐 **크림+골드테두리**로 최종 확정. 여러 탭의 저장·전송 버튼을 이 톤으로 **통일**.
- **버튼 라벨은 처음 보는 사람도 즉시 이해되는 문구로.** _(반려)_ "전체복사" 같은 모호어 지양 → 무슨 동작인지 분명하게("전체 저장 후 카톡 전송", "기록 저장").
- **나란히 놓인 두 버튼은 컨테이너 폭·크기를 100% 동일하게** 맞춘다(원장이 여러 번 강조).
- **정렬**: 상단바 저장/전송 버튼은 **우측 정렬**. 인라인 복사버튼은 없애고 상단 하나로 통합.
- **onclick 바인딩(치명적)**: 인라인 화살표함수 `{{ () => ... }}` 금지(버튼 먹통) → rv에 **핸들러 참조**를 만들어 `onclick="{{ navGoFee }}"`. (CLAUDE.md §3.)
- **숫자 입력은 스피너 화살표 제거**(타이핑만).

- **리포트/카드 섹션 테두리 = 얇은 골드 `1.5px solid gold-200` 통일**(출결·과제·시험성적·마일리지·성장추이 등 전 섹션).

## 6. 레이아웃 원칙

- **모든 탭 콘텐츠는 `<main class="scroll-y">` 안 `.main-pad` 래퍼 안에 있어야 함.** 밖에 있으면 빈화면/겹침. (CLAUDE.md §7-1.)
- **"한눈에 다 들어오게" — 스크롤 압박 반려.** 세로로 쭉 늘어선 리스트 대신 **좌우 2열·요약 카드**로 한 화면에. (선생님일지 주간·월간은 좌우 1:1 2열.)
- **전체 원생 보드(전체 원생/내 학생 한눈에) = 세로 1열 리스트 금지, "그룹 + 밀집 타일 그리드".** 학생 한 명당 가로 한 줄로 쌓으면 스크롤 압박이 심해 반려됨. 대신 **반별로 묶고**(반 헤더 = 반별 색점 + 인원 + "관리 N" 뱃지, §8 반별 색), **각 반 학생은 `repeat(auto-fill,minmax(160px,1fr))` 밀집 타일 그리드**(타일 = 등급색 좌측 3px 띠 + 이름 + 등급 배지 + 점수 + ▲▼델타 + 미니 추이 스파크라인, **관리 필요 학생은 붉은 타일 `#FDF7F6`**). **권한별 그룹 위계**: **학원일지(원장) = 선생님별 카드(이니셜·이름·N반·N명·관리필요 수) → 그 선생님 반별 → 학생 타일**(선생님은 `c.owner`, 미배정은 '원장'). **선생님일지 = 본인 담당 반별 → 학생 타일**(선생님 카드 없음). 필터칩(전체/관리필요/최근하락/미상담)은 그룹 안에서 적용되고, 해당 없는 반/선생님은 접힘. 타일 클릭 = 학생 종합 프로필(searchProfile). (구현: rv `boardGroups`, v32.399.) _(반려)_ 세로 리스트 1열.
- **빈 여백 금지 — 섹션이 늘거나 줄면 박스 높이가 자동 확장/축소**되어 빈 공간이 남지 않게(입력칸 자동확장 방식을 다른 탭에도).
- **콘텐츠 폭은 탭 간 통일(기준 `1060px`).** _(반려)_ 특정 탭만 폭이 넓거나 뒤로가기버튼·반선택바가 붙어 튀는 것. 세그먼트는 색만 바뀌고 내용만 교체(다른 세그먼트와 동일 로직·헤더).
- **홈(랜딩) 첫 화면 = "오늘 대시보드"**(인사 + 오늘 시간표 + 일지 바로가기 + 전달사항). _(반려)_ 랜딩이 바로 일간일지 입력칸인 것, 랜딩의 반 선택바.
- **홈 히어로 아래 세로 여백 넉넉히 — "여백의 미".**
- **바탕화면은 크림톤, 히어로만 딥그린**(대비로 우아하게).
- 스크롤바는 화면 오른쪽 끝에 붙게. 탭 전환 시 스크롤 초기화, 학생 종합 닫기 시 원래 탭·위치 복귀.

  - **Why:** "코멘트 / From your teacher / 담당 Benjamin" 후보를 비교한 끝에 선택. _(반려)_ "코멘트"는 "감성이 없다"(스타벅스 감성 아님)고 반려.
- **서명은 학생별 담당 선생님 이름으로 자동 삽입**(학생 입력 시 담당쌤 이름칸 값). _(반려)_ 우하단 고정 "담당 Benjamin 선생님".

- **캡처는 화면 밖 복제본(clone)을 캡처**(원본 캡처는 모달 잘림·미리보기 오염·stale 문제).

- **성적통지서 헤더 딥그린 = 로그인 히어로와 동일 그린으로 통일.** 딥그린 헤더 그라디언트는 반드시 로그인 게이트와 같은 `linear-gradient(168deg,#0e4a35 0%,#0a3a29 52%,#072017 100%)`를 쓴다. _(반려)_ 어두운 `linear-gradient(135deg,#0b3b2a→#00211a)`는 "#00211a(거의 검정)라 우중충"하다고 반려. **헤더 워터마크 = 실제 HIS 로고 PNG를 크게·저투명(흰 실루엣 `filter:brightness(0) invert(1);opacity:0.08`)으로** 우상단에 은은하게(골드 radial glow 대신). 헤더 하단 얇은 골드 헤어라인. 배정 밴드 딥그린 카드도 같은 로그인 그린으로 하모나이즈. **본문(성명표·점수표·도넛·레이더·진단·등급표)은 이미 크림/딥그린/골드 HIS 톤이라 유지.** (v32.402.)

## 8. 일정 · 시간표 (선생님 일지)

- **"종이 시간표처럼 전체를 한눈에".** 월(날짜) 형식은 취하되 **주간 시간표 형식으로 보이도록 주·월 통합**(가시성 우선). 요일은 **일요일까지**, **반마다 색 구분.**
- **월간 일정 = Google Calendar 스타일**: 유형별 색 칩, 다일 스팬바, 시간 표시, 컬러 보더, 오늘 날짜 초록 원.
- **다일(多日) 일정은 배경색이 날짜에 걸쳐 이어지게**(예: 시험기간 7.2~7.5 색 채움). **칩 클릭 = 삭제 아니라 수정**(구글 캘린더처럼).
- **이벤트 유형(색)**: 학교시험·모의고사·보강·상담(파랑)·학원·기타(이 순서). **색은 파스텔톤** — _(반려)_ 너무 짙은 색.
- **주간 시간표 = 반 블록을 "수업 시작~종료 시간만큼만" 세로 확장.** _(반려)_ 전체 시간축 길이 자체를 늘이는 것(전체가 한 화면에 보여야). 컴팩트(행높이 PPM 0.7), 이전/다음 주 네비.
- **학생 이름 폰트는 줄여 두 명씩 한 줄, 줄바꿈 금지.**
- **일정 입력 = 달력 날짜 숫자를 탭하면 그 자리에 인라인 폼.** _(반려)_ FAB·별도 큰 폼. 오전/오후 구분 없이 간단 입력, 날짜·기간·시간을 폼 맨 위 한 줄에.
- **반별 요일·시간 입력칸은 3반까지 한 줄 유지, 줄바꿈 금지.**
- 주간·월간 헤더 높이 통일(약 44px), 두 컬럼 1:1 폭 고정(`min-width:0; overflow:hidden`).
- **등하원 알림 톤(Apple×Starbucks, v32.409)**: ①**성장일지 등하원 스트립** — 캡처 리포트 '오늘 수업 내용' 위에 위아래 헤어라인(박스X)+`TODAY` 레터스페이스 라벨+탭formatting 시각(등원/하원)+출결 pill. 출결색 출석`#1F7A4D`/`#EAF3EA`·지각`#9A7B16`/`#FBF1D8`·결석`#B23A2C`/`#FBEAE5`·보강`#1F5C8A`/`#E7F0F6`. ②**미등원 안전망** — 선생님일지›등하원 상단, 크림카드+테라코타 좌측룰(`#C77A46`) 행, `SAFETY·실시간` 라벨, 딥그린 연락 pill. 실시간은 미등원만(등하원 '왔다갔다'는 성장일지로 충분). 이모지 대신 타이포·헤어라인.
- **키오스크 생일 축하 화면(v32.513, 원장 컨펌 «B — 크림 럭스»)**: 학생이 생일에 뒷4자리로 체크인하면 키오스크 전체 오버레이로 축하. 톤 = **크림(#F4ECD7) 바탕 + 얇은 골드 인셋 테두리(서식/스테이셔너리 느낌) + 실제 히즈 로고 워터마크(코너 블리드·저투명·살짝 회전, «감각있게»)**. 구성 위→아래: 골드 데이트라인(`2026 · 07 · 09`, 레터스페이스) → *Happy Birthday*(골드 세리프 이탤릭) → 골드 헤어라인 → **「{호격}, 생일 축하해」**(딥그린 대형, 호격=이름 받침 따라 아/야 자동 `vocName`) → "히즈어학원이 {이름} 학생의 하루를 축하합니다" → **딥그린 pill 「{등원/하원} 완료 · 시각」**(골드 체크). **담당쌤 서명 제외**(원장 지시). 크기는 **vmin 기반 clamp**로 세로(LG 스탠바이미)·가로 양쪽 대응. _(반려)_ 플로팅 크림 박스를 딥그린 위에 얹기·클립아트 캔들 이모지·담당쌤 서명. **과정: A(키노트 몰입)/B(크림 럭스)/C(에디토리얼) 3안 시안 → B 채택 → 로고 워터마크·서명제외 반영 → 컨펌 → 구현**(§14 ① 선-위젯-컨펌 준수). 발동조건: 상담·진단 «생일»칸(`intake.birth`, yyyy.mm.dd) 입력 필요.
- **선생님 일지 = 애플식 세그먼트 4개(`scheduleSeg`): 「내 학생」(기본)·「시간표」·「등하원」·「보고」.** 세그먼트 바를 뷰 맨 위에 고정, 색만 바뀌고 내용만 교체(§6 원칙). 각 세그먼트가 하나의 콘텐츠만 노출(스크롤 압박 제거) — 내 학생=반별 보드(KPI+타일, 본인 담당 반만), 시간표=주간+월간, 등하원=체크인, 보고=원장님께 보고·건의. rv 플래그 `isSchedStu/isSchedJournal(='cal')/isCheckin/isSchedTalk`, 기본값 `S.scheduleSeg||'stu'`. (v32.408.) ⚠️ 게이팅 sc-if 경계는 **div 균형**으로 잡을 것 — 보드 안에도 `var(--surface-card)` 카드가 있어 앵커 검색이 보드 내부에 오걸림(초기 버그).

## 9. 입력 · 폼 UX 패턴

- **원터치 칩 버튼 우선**: 출결·과제 등은 드롭다운 대신 칩(attBtns/hwBtns). 가로 한 줄 고정, 줄바꿈 금지.
- **인라인 편집 우선**: 시험명 등은 별도 상단 편집란 없이 카드에서 직접.
- **반배정 대기(신규생 홀딩) 행 = 한 줄 고정(줄바꿈 금지) + 이름 인라인 편집 + 삭제 휴지통.** 이름칸은 클릭해 바로 수정(임시 이름 "신규생"→실명), 우측 끝 휴지통(붉은 SVG)은 confirm 후 삭제(학생 + 그 학생의 상담·시험 기록까지 제거, id 고유해 타 학생 무영향). (구현: rv row `onRename`/`onDelete`, v32.400.) 앱엔 아이콘 폰트가 없으니 **아이콘은 인라인 SVG로** 그림(Tabler 등 웹폰트 사용 불가).
- **textarea 기본 2줄 → 자동확장**(학생 한 명 입력칸 전체가 한눈에). _(반려)_ 기본 3줄.
- **위험/저빈도 버튼은 없앤다**: 마일리지 전체 초기화·오늘 기록 삭제·이전회차 불러오기 등. 자동저장이라 불러오기 불필요, 초기화는 오클릭 위험(필요 시 콘솔 경로만).
- **누계값(마일리지 등)은 직접 입력·수정 가능**해야 함 — 학생마다 시작값이 달라 0부터가 아님(자동적립 + 수동입력 공존).

## 10. 장학마일리지 안내 (설명용 모달)

- **명칭 = "히즈 장학금마일리지"** ('장학**금**마일리지' — '금' 반드시 포함). 서사 = **"성실함은 기록되고, 기록은 티어가 된다".**
- **긍정·게임 서사 강조**: "금방 도달할 수 있고 손해 아닌" 느낌, "하루만 성실해도 +1.0". **벌점(결석 0·과제 −0.2/−0.1)은 두되 강조하지 말 것.**
- **히어로 = 딥그린 그라디언트 + HIS 워터마크**(로그인 좌측과 동일 톤), 워터마크 우상단.
- **레벨 차트 = 실제 티어 엠블럼(SVG) + 현금 함께**, 최고 보상(그랜드→…→씨앗) **역순 노출**, 금액은 **`₩` 기호 + 000까지 전부 표기**(약어 금지), 등폭 정렬. 모든 박스 얇은 금색 테두리, 좌측정렬 스펙시트(애플 톤).
- 캡처 버튼은 전용 핸들러(성공/실패 피드백을 모달 상단에 직접 표시).

## 11. 마일리지 규칙(수치) — 참고

- 출결 +0.2 / 시험통과 +0.5 / 과제: 완료 +0.4·일부 −0.1·미완료 −0.2·미출제 0 / 결석 0. 음수 방지.
- 장학금 티어별 상향 확정치(마스터 ₩250,000 … 그랜드 300개 ₩1,000,000). 변경 시 `scholarshipAmt` 계산식 + 안내 표시 **양쪽** 반영.

## 12. 검증 규칙 (디자인 작업 필수 절차)

- **디자인/UI를 바꾸면 반드시 렌더링 → 스크린샷으로 직접 눈으로 확인한 뒤** 원장에게 보여줄 것. 눈으로 안 보고 만들면 어긋남.

- **사용자에게 "테스트해보세요"로 검증을 미루지 말 것** — 내가 먼저 확인.

## 13. 명명 · 카피 · 페르소나 톤

- **디자인 페르소나 = "애플·스타벅스 수석 디자이너"**(정제·고급·심미). **상담/코칭 카피 = "오은영(정서) + 입시전문가"**(‘검사’ 아니라 ‘관심’으로 느껴지게).
- **카피는 광고사 헤드카피처럼 한 줄로 함축.** _(반려)_ "매일의 성실함이 마일리지가 되고, 레벨이 오르고, 장학금까지" 같은 나열형 부제, "공부=현금" 직결 문구.
- **홈 인사 = "좋은 하루예요. {원장님 이름} 원장님" 톤.**
- 명칭 통일: "주간 시간표→주간 일정", "월간 달력→월간 일정", "반별 시간 설정→반별 일정".
- 학생 리포트: "~이의 성장일지", 서명 "— from {담당쌤}".

## 14. 협업 프로세스 (원장이 가장 반복해서 강조한 3원칙) — 디자인 작업 시 최우선

> 디자인 "결과물" 규칙은 아니지만, **디자인 작업을 어떻게 진행할지**에 대한 원장의 반복된 강한 요구라 최상위로 지킵니다. 커밋·기존 문서 어디에도 없던, 원장의 최대 스트레스 지점.

- **① 선(先)위젯-컨펌**: 새 UI는 **먼저 시안(위젯/이미지)으로 보여주고 원장이 컨펌한 뒤** 코드에 반영. _(반려)_ 컨펌 전 임의 구현. ("내가 컨펌하기 전까지는 생성하지마"를 수십 회.)
- **② 시킨 것만 정확히**: **명령하지 않은 기존 기능은 절대 손대지 말 것**(연쇄 오류의 최대 원인). 특히 카톡 캡처 이미지는 그대로 보존. ("시키지도 않은 걸 건들어서 기존 기능까지 마비.")
- **③ 요청 시 즉시 원복**: "되돌려/별로야"라고 하면 새 실험안을 **즉시 이전안으로 복원**. 원복이 기본값.
- **④ 역할별 정보 노출 분리(권한=디자인)**: 선생님에겐 수납·미납 숨김, 원장에겐 선생님별 전달사항 노출. 랜딩도 역할별로 다르게.

## 부록 C. 이 지침서의 출처와 범위 (정직한 한계)
