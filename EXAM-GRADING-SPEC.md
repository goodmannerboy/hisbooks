# 문항별 자동채점·유형분석 시스템 — 사양서 (EXAM-GRADING-SPEC.md)

> 원장 요청(2026-07-04): 엑셀 `모의고사성적표_고등.xlsx`의 워크플로를 앱에 애플식으로 리뉴얼.
> **정답키·배점·유형은 한 번만 세팅 → 학생 답만 입력 → 총점·등급·유형별 정답률 자동 채점 → 알림장 자동 생성.**
> 성적 성장일지(모의고사)와 신규 레벨테스트 **양쪽 동시 적용**. HIS 통지서 톤(v32.402 헤더) 재사용.

## 확정 결정 (원장)
- **등급 = 원점수식(엑셀 그대로)**: 90↑=1등급, 80~89=2, 70~79=3, 60~69=4, 50~59=5, 40~49=6, 30~39=7, 20~29=8, 10~19=9, 10 미만=10등급. (총점 100 기준. 배점 합계가 100이 아니면 100 환산 후 적용.)
- **적용 범위 = 성적성장일지 + 레벨테스트 동시.**
- **유형 = 프리셋 고정 + 정답만 입력.** 시험마다 정답(1~5)·배점만 입력, 유형은 프리셋에서 선택/자동.

## 유형 프리셋 (고등, 엑셀 기준)
대분류 → 세부유형:
- **듣기** (1~17): 세부유형 없음(듣기).
- **중심 내용 파악**: 글의 목적 / 심경·분위기 / 필자의 주장 / 함축의미 추론 / 글의 요지 추론 / 글의 주제 추론 / 글의 제목 추론.
- **세부 정보 파악**: 도표 내용 이해 / 내용일치 / 실용문 / 어법 / 어휘.
- **빈칸 추론**: 빈칸추론.
- **간접 쓰기**: 흐름과 무관한 문장 / 글의 순서 / 문장 삽입 / 요약문 완성.
- **장문 독해**: 글의 제목 / 어휘 추론 / 글의 순서 / 지칭 추론 / 내용일치 불일치.

(중등/레벨테스트용 프리셋은 별도 — 초기엔 고등 프리셋 재사용, 추후 원장 확인 후 중등 세트 추가.)

## 데이터 모델 (index.html data blob)
```
data.examSets = [ {
  id, name, date, grade,            // 예: '2026 6월 모의고사', '고3'
  questions: [ { no, key(1..5), pt, group, type } ],   // 정답키 그리드
  totalPt,                          // = sum(pt)
  submissions: { [studentId]: { answers: { [no]: 1..5 }, updatedAt } }
} ]
```
채점은 렌더 시 계산(파생값): `score = Σ(answers[no]===key ? pt : 0)`, `pct = round(score/totalPt*100)`, `등급 = gradeOf(pct)`, `유형별 정답률 = 맞은수/문항수 by group`.

## 화면 (애플식, HIS 톤)
1. **시험 세팅(정답키)** — 문항 그리드: 번호 · 정답(1~5 칩) · 배점 · 유형(프리셋 드롭다운). 45문항 한 화면, 붙여넣기/일괄 지원 고려.
2. **학생 답 입력(빠른 채점)** — 학생 선택 → 문항별 1~5 탭(OMR 스타일) → 즉시 O/X·총점·등급·정답수 실시간. 반 전체를 한 화면에서 순차 처리.
3. **알림장(자동)** — 총점·등급(원점수식) + **유형별 정답률(약점 붉게)** + 문항별 O/X, HIS 통지서 톤(딥그린 헤더+로고 워터마크) → 카톡 이미지 복사(기존 captureEl 재사용).

## 성장일지 연동
- 자동 채점 결과(score/total=100/grade)를 기존 `data.exams` 레코드로 저장 → 성적 성장일지 추이·리포트에 그대로 반영.
- 유형별 정답률은 examSets에 파생 저장(약점 유형 상담·리포트 자동 노출).
